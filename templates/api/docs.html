Documentacao da API

Endpoints principais em JSON. Todos exigem CSRF/credenciais conforme configuracao.

Pedidos

  • POST /api/pedidos/criar – cria pedido rápido. Body: {itens:[{produto_id, quantidade}]}. Resposta: {success, pedido_id}
  • POST /api/pedidos/<id>/finalizar – finaliza com pagamento. Body: {metodo_pagamento, valor_recebido}. Resposta: {success, total}
  • GET /api/pedidos/em-aberto – lista pedidos abertos.
  • GET /api/pedidos/<id>/detalhes-json – detalhes completos do pedido.
  • POST /api/pedidos/<id>/adicionar – adiciona itens. Body: {itens:[...]}.

Estoque

  • GET /api/estoque/analytics – métricas de estoque.
  • POST /movimentacoes/nova – movimentação manual (form-data); ver telas internas.

Dashboard / RH

  • GET /api/dashboard/analytics?inicio=YYYY-MM-DD&fim=YYYY-MM-DD – métricas de vendas.
  • GET /api/rh/analytics – indicadores de RH.

Estrutura de resposta padrão: { success: bool, message: str, data?: any, code?: str }.
